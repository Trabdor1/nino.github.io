Procédure technique

Mise en place des VLAN et du routage inter-VLAN dans le contexte M2L

Option SISR – Solutions d’Infrastructure, Systèmes et Réseaux

Cette procédure présente la mise en place des VLAN et du routage inter-VLAN dans le contexte de la Maison des Ligues de Lorraine (M2L). L’objectif est de segmenter logiquement le réseau, d’isoler les flux selon leur usage et de permettre la communication entre les sous-réseaux lorsque cela est nécessaire.

1. Présentation du projet

1.1 Contexte

Dans le cadre du contexte réseau de la Maison des Ligues de Lorraine (M2L), l’infrastructure doit être segmentée en plusieurs réseaux logiques afin de séparer les ligues, les services internes et les équipements spécifiques comme la téléphonie IP. Cette séparation permet d’améliorer la sécurité, la gestion du trafic et l’administration de l’ensemble du réseau.

Dans ma maquette Packet Tracer, les postes utilisateurs des ligues Karaté, Athlé et Yoga sont répartis dans des VLAN différents. Les téléphones IP sont isolés dans un VLAN spécifique dédié à la voix. Le trafic des ligues est routé par deux routeurs Cisco puis transite par un switch central nommé SWDMZ, qui sert de point de passage vers le reste de l’infrastructure.

1.2 Objectifs

  • Segmenter le réseau des ligues en plusieurs VLAN.
  • Isoler les téléphones IP dans un VLAN VOIP.
  • Attribuer une passerelle à chaque réseau logique.
  • Mettre en place le routage inter-VLAN grâce aux routeurs Cisco.
  • Faire transiter les communications via le switch central SWDMZ.
  • Vérifier le bon fonctionnement par des tests de connectivité.

1.3 Prérequis techniques

  • Une maquette Packet Tracer déjà câblée
  • Deux routeurs Cisco pour les ligues
  • Un switch central SWDMZ
  • Des switchs d’accès pour les ligues
  • Des postes clients et téléphones IP
  • Le plan d’adressage IP défini dans le cahier des charges M2L

2. Préparation des équipements

2.1 Configuration de base des routeurs

enable
configure terminal
hostname RTLIGUES1
no ip domain-lookup
enable
configure terminal
hostname RTLIGUES2
no ip domain-lookup

2.2 Configuration de base du switch central

enable
configure terminal
hostname SWDMZ
no ip domain-lookup

2.3 Explication

La commande enable permet d’accéder au mode privilégié. La commande configure terminal permet d’entrer dans le mode de configuration globale. La commande hostname permet d’identifier clairement l’équipement dans la maquette. Enfin, no ip domain-lookup désactive la résolution DNS automatique et évite les temps d’attente lorsqu’une commande est mal saisie.

3. Mise en place de la segmentation réseau

3.1 Répartition des équipements dans les VLAN

  • VLAN 10 – Karaté : PC0 et PC1
  • VLAN 11 – Athlé : PC2 et PC3
  • VLAN 12 – Yoga : PC4 et PC5
  • VLAN 5 – VOIP : IP Phone0 et IP Phone1

3.2 Création des VLAN sur les switchs des ligues

enable
configure terminal

vlan 10
name KARATE

vlan 11
name ATHLE

vlan 12
name YOGA

vlan 5
name VOIP

end
write memory

3.3 Explication

Chaque VLAN correspond à un domaine de broadcast distinct. Cette étape permet donc de séparer logiquement les machines sans multiplier les équipements physiques. La commande write memory permet d’enregistrer la configuration.

3.4 Affectation des ports d’accès

interface fastEthernet0/5
switchport mode access
switchport access vlan 10
spanning-tree portfast

3.5 Explication

La commande switchport mode access définit un port destiné à un seul VLAN. La commande switchport access vlan 10 associe ce port au VLAN 10. Enfin, spanning-tree portfast accélère la mise en service du port pour un équipement terminal.

3.6 Mise en place des trunks

interface gigabitEthernet0/1
switchport mode trunk
switchport trunk allowed vlan 5,10,11,12

3.7 Explication

Le trunk permet de faire circuler plusieurs VLAN sur une seule interface physique grâce à l’encapsulation 802.1Q. Cette configuration est nécessaire pour le transport des VLAN entre les switchs et les routeurs.

4. Configuration du routage inter-VLAN

4.1 Principe retenu

Le routage inter-VLAN est assuré par deux routeurs Cisco à l’aide du principe de router-on-a-stick. Chaque routeur utilise une interface physique reliée au switch des ligues, puis des sous-interfaces correspondant aux différents VLAN.

4.2 Configuration du routeur RTLIGUES1

enable
configure terminal
hostname RTLIGUES1
no ip domain-lookup

interface gigabitEthernet0/1
no shutdown

interface gigabitEthernet0/1.10
encapsulation dot1Q 10
ip address 172.16.10.254 255.255.255.0

interface gigabitEthernet0/1.11
encapsulation dot1Q 11
ip address 172.16.11.254 255.255.255.0

interface gigabitEthernet0/0
ip address 10.10.10.1 255.255.255.252
no shutdown

ip route 172.16.12.0 255.255.255.0 10.10.10.2
ip route 172.16.5.0 255.255.255.0 10.10.10.2
ip route 172.16.2.224 255.255.255.224 10.10.10.2
ip route 192.168.0.0 255.255.255.248 10.10.10.2
ip route 192.168.1.0 255.255.255.248 10.10.10.2
ip route 0.0.0.0 0.0.0.0 10.10.10.2

end
write memory

4.3 Explication

Les sous-interfaces permettent de traiter plusieurs VLAN sur une seule interface physique. La commande encapsulation dot1Q associe chaque sous-interface à son VLAN. Les adresses IP configurées représentent les passerelles par défaut des réseaux correspondants. Les routes statiques permettent d’atteindre les réseaux non directement connectés.

5. Configuration du second routeur des ligues

5.1 Configuration du routeur RTLIGUES2

enable
configure terminal
hostname RTLIGUES2
no ip domain-lookup

interface gigabitEthernet0/1
no shutdown

interface gigabitEthernet0/1.12
encapsulation dot1Q 12
ip address 172.16.12.254 255.255.255.0

interface gigabitEthernet0/1.5
encapsulation dot1Q 5
ip address 172.16.5.254 255.255.255.0

interface gigabitEthernet0/0
ip address 10.10.10.5 255.255.255.252
no shutdown

ip route 172.16.10.0 255.255.255.0 10.10.10.6
ip route 172.16.11.0 255.255.255.0 10.10.10.6
ip route 172.16.2.224 255.255.255.224 10.10.10.6
ip route 192.168.0.0 255.255.255.248 10.10.10.6
ip route 192.168.1.0 255.255.255.248 10.10.10.6
ip route 0.0.0.0 0.0.0.0 10.10.10.6

end
write memory

5.2 Explication

Le principe est identique à celui de RTLIGUES1. Chaque sous-interface représente un VLAN et chaque adresse IP configurée devient la passerelle des équipements du réseau correspondant.

6. Configuration du switch central SWDMZ

6.1 Rôle du switch central

Le switch SWDMZ sert de point de transit entre les routeurs des ligues, le réseau M2L et les équipements de sécurité. Il ne réalise pas de routage IP mais assure le transport des VLAN nécessaires.

6.2 Création des VLAN de transit

enable
configure terminal
hostname SWDMZ
no ip domain-lookup

vlan 2
name INFORMATIQUE_M2L

vlan 100
name TRANSIT_RTL1

vlan 101
name TRANSIT_RTL2

6.3 Configuration des ports

interface fastEthernet0/24
switchport mode access
switchport access vlan 100

interface fastEthernet0/1
switchport mode access
switchport access vlan 101

interface fastEthernet0/2
switchport mode trunk
switchport trunk allowed vlan 2

interface gigabitEthernet0/1
switchport mode trunk
switchport trunk allowed vlan 2,100,101

interface gigabitEthernet1/1
switchport mode access
switchport access vlan 2

6.4 Configuration de l’adresse de gestion

interface vlan 2
ip address 172.16.2.253 255.255.255.224
no shutdown

ip default-gateway 172.16.2.254

end
write memory

8. Vérification de la configuration

8.1 Vérifications sur les switchs

show vlan brief
show interfaces trunk
show running-config

8.2 Vérifications sur les routeurs

show ip interface brief

show running-config

9. Tests de connectivité

9.1 Tests à effectuer

ping 172.16.10.254
ping 172.16.11.10
ping 172.16.12.10
ping 172.16.12.254
ping 172.16.2.242

9.2 Explication

Ces tests permettent de vérifier que la passerelle de chaque VLAN est joignable, que le routage inter-VLAN fonctionne correctement et que les équipements peuvent communiquer avec le réseau M2L.

10. Résultat obtenu

À l’issue de cette mise en œuvre, les réseaux des ligues ont été correctement segmentés dans des VLAN distincts. Les téléphones IP ont été isolés dans un VLAN spécifique dédié à la téléphonie. Les deux routeurs ont été configurés avec des sous-interfaces 802.1Q afin d’assurer le routage inter-VLAN, tandis que le switch central SWDMZ a servi de point de transit vers le reste de l’infrastructure.

Cette réalisation m’a permis de reproduire une architecture réseau structurée, cohérente avec le contexte M2L, et de mettre en pratique la segmentation logique, le routage inter-VLAN et la configuration d’équipements Cisco dans un environnement simulé.
